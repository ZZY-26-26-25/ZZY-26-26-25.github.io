# AI 智能体大师课

面向零基础学习者的免费交互式课程。课程以“旅行规划智能体”为贯穿案例，从 AI 与 LLM 基础逐步推进到工具调用、资料检索、RAG、记忆、规划、多智能体、评测、安全、产品化和长期运营。

## 理论内容的来源原则

优先级依次为：

1. 正式规范、标准和政府/学术机构文件；
2. 原始论文及作者项目页；
3. 协议、框架或平台官方文档；
4. 公认教材或原作者技术说明。

厂商宣传、排行榜、社交平台和二手教程不能单独支撑能力、价格、安全或效果声明。协议、产品、法律和价格等时变信息必须注明核验日期；无法公开核验的内容明确标记为未知。

旅行案例同样遵守来源分层：社交平台只用于发现摄影机位与体验线索，开放状态、合法性、安全、交通和天气等关键事实必须由官方或一手来源核验。

## 数据与安全边界

学习进度默认只保存在访问者当前浏览器的 `localStorage`，网站不上传这些数据。公开仓库不会保存 API Key、私有旅行数据或机构内部凭据。

GitHub Pages 只能承载公开静态前端，不能安全保存长期服务端密钥、代理用户账号或承担可恢复后台任务。未来真实实验只会从学习者控制的 Secret 或受控后端读取凭据。

## 内容构建

模块理论稿位于 `content/theory-*.json`，按 `THEORY_STANDARD.md` 编写，并随网站源码一起版本化。合并为公开数据：

```bash
node scripts/build-theory.mjs
```

构建脚本要求：

- 10 模块合计恰好 51 课；
- 每课至少 4 个目标、4 个理论章节、3 个误区、1 个旅行案例、3 个自检题和 3 个主要来源；
- 所有来源使用 HTTPS；
- 不包含私有端点或疑似密钥。

## 发布前校验

```bash
node --check app.js
node --check course-data.js
node scripts/validate-course.mjs
```

校验覆盖课程数量、ID 与顺序、正文长度、来源字段、弱来源拦截、缓存版本、Notebook 输出清理和密钥扫描。
