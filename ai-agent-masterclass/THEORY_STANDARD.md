# 《AI 智能体大师课》理论课程编写与审校标准

版本：1.1  
适用阶段：51 节核心课的理论建设期

## 1. 当前阶段的目标

本阶段只建设课程的理论主干。目标不是把术语铺得越多越好，而是让零基础学习者形成一套能迁移到不同模型、框架和平台的心智模型。

现有 Colab、Notebook 与模型 API 实验保留，但不再决定课程顺序。后续实践必须反过来验证已经稳定的理论结构，不能为了展示某个工具而临时改写原理。

## 2. 51 课的知识主线

课程保持以下依赖关系：

1. AI、LLM、工作流与智能体的边界；
2. 数字系统、LLM 与接口的基础概念；
3. 单智能体、工具契约与控制循环；
4. 资料检索、证据管理与 RAG；
5. 状态、上下文与长期记忆；
6. 规划、工作流与长任务；
7. 多智能体与互操作协议；
8. 评测、可观测性与安全；
9. 产品架构、可靠性、成本与人机交互；
10. 从需求到运营证据的毕业综合方法。

“旅行规划智能体”只作为贯穿案例，用来暴露时间敏感信息、工具权限、隐私、风险和多方案决策等真实问题；它不能替代理论本身。

## 3. 每课必须回答的八类问题

每一课必须同时具备：

1. **学习目的**：这个概念解决什么问题，不学会导致什么错误；
2. **核心定义**：与相邻概念怎样区分，定义采用什么范围；
3. **工作机制**：输入、处理、状态变化和输出之间的因果关系；
4. **适用边界**：它能解决什么，不能保证什么；
5. **关键权衡**：质量、成本、延迟、复杂度、隐私或风险怎样互相影响；
6. **常见误解**：至少纠正三个初学者容易形成的错误心智模型；
7. **案例映射**：把理论映射到旅行规划智能体中的一个真实决策；
8. **来源与自检**：给出可追溯来源，并用理论问题检查是否真正理解。

## 4. 统一课程结构

每课使用结构化数据保存，最少包含：

- 4–6 个可解释的学习目标；
- 明确的先修知识；
- 至少 4 个理论章节，每章不少于 2 个完整段落；
- 至少 3 组“误解—纠正”；
- 一个包含决策规则的旅行案例；
- 4–6 条本课结论；
- 3 个带参考答案的理论自检题；
- 至少 3 个逐条注明用途的主要来源；只保留直接支撑正文的来源，不设机械上限。

理论自检可以要求比较、解释因果或判断边界，但本阶段不要求写代码、配置平台或调用真实 API。

## 5. 来源层级

课程按以下优先级选择资料：

1. 规范、标准与政府/学术机构发布的正式文件；
2. 原始论文及其正式项目页；
3. 协议、框架或平台的官方文档；
4. 公认教材或原作者的技术说明；
5. 其他资料只可作为线索，不作为关键事实的唯一证据。

厂商宣传、排行榜、社交媒体和二手教程不能单独支撑能力、价格、安全或效果声明。涉及版本、价格、协议、法律或产品能力的内容必须注明核验日期；无法公开核验的内容必须明确写成“未知”或“待验证”。

## 6. 准确性规则

- 不把教学类比当成严格分类。例如，“LLM 像大脑”只能帮助理解分工，不能推出意识、意图或责任能力。
- 不把课程自定义框架冒充行业标准。例如，八要素检查表和自主度分级都要注明是教学工具。
- 不把模型的流畅输出当成事实正确，也不把一次成功运行当成系统有效。
- 不把上下文、会话状态、任务状态和长期记忆混为一谈。
- 不把 OpenAI-compatible、开放权重、开源 AI、免费额度和可商用混为一谈。
- 不把工具可访问等同于工具可执行；副作用、权限和审批必须单独判断。
- 不把 RAG、长上下文、微调和记忆视为可以互相任意替换的“增强按钮”。
- 不把更多智能体、更长 Prompt 或更多自主权默认视为更好。
- 不公开要求模型暴露私有思维过程；课程只依赖可观察的输入、动作、结果和验证证据。

## 7. 理论完成度门槛

一课只有同时通过以下检查，才标记为“理论已发布”：

- 术语与前后课一致，先修关系成立；
- 机制解释包含因果链，而非定义堆叠；
- 关键边界和失败模式已经写明；
- 旅行案例没有偷渡尚未讲授的能力；
- 每个可变事实有权威来源或明确的核验边界；
- 所有链接可访问，来源说明与正文主张一致；
- 内容校验脚本通过；
- 桌面端、移动端、键盘导航和搜索均可使用。

## 8. 实际教学验收

“理论已发布”只说明内容结构与来源通过门禁，不等于学习者已经被教会。课程进入实际教学后，每课还要观察：

- 零基础学习者能否先从具体问题进入，再逐步命名抽象概念；
- 能否不照抄原文，用自己的话解释核心机制；
- 能否指出至少一个适用边界或常见误解；
- 能否把理论迁移到旅行规划智能体中的一个新判断；
- 哪些疑问反复出现，是否暴露了先修缺口、术语过密或来源错配。

站点分别记录“尚未判断”“有疑问，待复习”和“已确认理解”。打开页面、滚动到底或点击下一课都不自动等同于理解；有疑问的学习者可以继续浏览，但该课会进入待复习状态。所有个人回答与疑问仅保存在学习者当前浏览器，不进入公开课程源码。

## 9. 后续实践如何接回理论

理论稳定后，每课再按同一顺序增加实践层：

1. 手工或纸面模拟；
2. 最小可运行实验；
3. 故障注入与调试；
4. 量化或规则化评测；
5. 旅行规划智能体的版本增量。

实践不会改写理论编号，而是作为 `LAB` 与对应理论课关联。这样 GitHub 网站承担教材主线，Colab 等平台承担可替换的实验环境，模型 API 只是实验资源之一。
